## PART 1 — THE GAME

### What We're Building
A RuneScape-inspired fantasy RPG on Roblox with deep crafting, race-based progression,
magic schools, PvP wars, and a reincarnation system. Launching as a [BETA] — playable
while still being built. Inspired by RuneScape, anime like That Time I Got Reincarnated
as a Slime, and our family's imagination.

## RACES — Tiered by Rarity
Players roll for their race at character creation. Rarer = lower chance.

| Tier       | Races                                      |
|------------|--------------------------------------------|
| Common     | Human, Elf, Dwarf, Orc                     |
| Uncommon   | Merfolk / Aquatic                          |
| Rare       | Vampire, Werewolf, Draconic                |
| Legendary  | Demigod, Angel, Demon                      |
| Secret     | Golden Merfolk (unlocked by beating the Leviathan boss) |

Each race has natural skill affinities:
- Dwarf — faster mining and smelting XP
- Elf — stronger magic, better spell accuracy
- Angel / Demigod — Holy Magic affinity, healing power
- Draconic — Fire Magic affinity, natural armor defense
- Vampire — Dark Magic, life steal
- Werewolf — attack speed bonus, beast instinct
- Merfolk — Water Magic affinity, aquatic abilities

## THE CRAFTING LOOP (Heart of the Game)

```
MINE ORE → SMELT INTO BAR → FORGE WEAPON/ARMOR → SOCKET GEMS → ENCHANT
```

Skill requirements at every step:
- High Mining level to extract rare ore
- High Smelting level to process it
- High Smithing level to forge it
- High Defense level to WEAR the armor
- High Attack level to WIELD the weapon

You cannot shortcut progression. You earn every piece of gear.

## PART 2 — THE DEVELOPMENT PLAN

### The Language: Lua
Roblox games are written in Lua — a lightweight scripting language.
It reads almost like plain English, making it beginner-friendly.

Example of what Lua looks like:
```lua
-- Give a player XP and check if they level up
local function addXP(player, amount)
    local stats = player:FindFirstChild("Stats")
    stats.XP.Value = stats.XP.Value + amount

    if stats.XP.Value >= stats.XPNeeded.Value then
        stats.Level.Value = stats.Level.Value + 1
        stats.XP.Value = 0
        print(player.Name .. " leveled up!")
    end
end
```

---

## HOW ROBLOX SCRIPTS WORK — The 3 Types

| Script Type    | Where It Runs         | What It Does                                      |
|----------------|-----------------------|---------------------------------------------------|
| ServerScript   | Roblox's servers      | Game logic, security, saving — players can't touch this |
| LocalScript    | Player's device       | UI, visual effects, input — what the player sees  |
| ModuleScript   | Called by either      | Shared data and functions — the rulebook          |

Think of it like a restaurant:
- ModuleScript = the recipe book (the rules everyone follows)
- ServerScript = the kitchen (does the real work, hidden from customers)
- LocalScript = the dining room (what the customer sees and touches)

## BUILD ORDER — Systems We'll Code One at a Time

### Step 1 — Race & Class Data Module (ModuleScript)
The rulebook. Defines every race, class, and their stats.
Nothing else works without this existing first.
```
RaceData: { name, rarity, rollChance, statBonuses, magicAffinity }
ClassData: { name, baseStats, startingAbilities }
```

### Step 2 — Character Stats System (ServerScript)
When a player joins, give them a stats object with:
```
Level, XP, XPNeeded
Health, Attack, Defense
MiningLevel, SmeltingLevel, SmithingLevel
MagicSchool, MagicAptitude
Race, Class
```

### Step 3 — Race Roll System (ServerScript + LocalScript)
- Player opens UI and hits Roll
- Server calculates result based on rarity weights
- Animation plays, race is revealed
- Connects to Reincarnation God NPC for Robux-based rolls

### Step 4 — XP & Leveling System (ServerScript)
- Combat, mining, and smelting all award XP to separate skills
- Each skill levels up independently (RuneScape style)
- Level gates what gear you can use

### Step 5 — Mining System (ServerScript + LocalScript)
- Ore nodes placed in the game world
- Player clicks node, server checks Mining level, grants ore if qualified
- Rare nodes respawn on timers
- Dwarf race passive gives faster gathering speed

### Step 6 — Smelting & Smithing System (ServerScript)
- Furnace object: bring ore + coal, smelt into bars
- Anvil object: bring bars, forge into weapons and armor
- Multi-metal combining logic: mix metals for unique results

### Step 7 — Gem Socketing & Enchanting (ServerScript)
- Enchanting table object in the world
- Player brings gear + gem, server applies stat bonus to item data
- High-tier metals allow multiple gem slots

### Step 8 — Combat System (ServerScript)
- Click to attack with cooldown timer
- Damage = Attack stat + weapon bonus + magic bonus
- Abilities tied to class and magic school
- Body enhancement magic = temporary stat buffs during combat

### Step 9 — PvP Zone System (ServerScript)
- Zones tagged in the world map
- Server checks if player is in a PvP zone
- Enables or disables PvP damage accordingly
- Better loot tables and XP multipliers active in PvP zones

### Step 10 — Quest System (ModuleScript + ServerScript)
- Quest data stored in a module (objectives, rewards)
- Server tracks each player's quest progress
- Completion triggers XP reward, item reward, or magic unlock

### Step 11 — War System (ServerScript)
- Guild and team creation
- War declaration system
- Large area combat with kill/death tracking
- XP multipliers for war participation
- Solo war attempts allowed with higher reward scaling

### Step 12 — Boss System (ServerScript)
- Boss AI with multiple phases
- Damage shared across the entire team
- Leviathan boss unlocks Golden Merfolk race on defeat
- Rewards distributed to all participants on victory

### Step 13 — Monetization (ServerScript)
- Roblox MarketplaceService handles all Robux transactions
- Game Passes: checked on join, perks applied (2x XP, VIP, cosmetics)
- Developer Products: repeatable purchases (extra rolls, boosts)
- All handled server-side so it cannot be cheated by players

### Step 14 — Data Saving (ServerScript)
- Roblox DataStore saves every player's progress permanently
- Saves on logout and periodically during play
- Loads on join so your character is always where you left it

## WHAT YOU'LL LEARN ALONG THE WAY

As we build each system together, you will naturally pick up:

| Concept                    | Where You'll Use It                        |
|----------------------------|--------------------------------------------|
| Variables and data types   | Storing player stats, ore names, levels    |
| Functions                  | Reusable actions like addXP, rollRace      |
| If/else statements         | Checking if a player is strong enough      |
| Loops                      | Checking all players, respawning ore nodes |
| Tables                     | Storing all race data, inventories, quests |
| Events                     | Responding to clicks, deaths, purchases    |
| Client-server communication | The most important Roblox concept          |

Each system we build, I will explain what it does and WHY it is written
that way — not just hand you finished code. This is your game and your skill to build.

## SUGGESTED STARTING POINT
Build the RaceData module first.
It is pure data with no complex logic, and writing it will teach you
how tables work in Lua — the single most used skill in all of Roblox development.

That one file becomes the foundation that every other system in the game plugs into.
